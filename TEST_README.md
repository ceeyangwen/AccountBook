
# 账户余额计算测试说明

## 问题修复

### 修复的问题
**信用卡支出余额计算错误**：之前使用信用卡支出20元时，余额反而减少了20元，正确逻辑应该是余额增加20元（负债增加）。

### 修复方案
1. 创建了独立的 `balanceCalculator.js` 模块，统一管理余额计算逻辑
2. 修复了负债/信用类账户的计算逻辑，对于普通支出/收入操作需要取反处理
3. 更新了 `app.js` 中的 `updateAccountBalance` 函数，使用新的计算模块

## 如何运行测试

### 前置条件
- 安装 Node.js (推荐 v14 或更高版本)

### 运行测试
```bash
node test/balanceCalculator.test.js
```

## 测试覆盖范围

### 1. 账户类型识别
- 现金、储蓄卡 → 资产类
- 信用卡 → 信用类
- 负债账户 → 负债类

### 2. 资产类账户计算
- 支出：余额减少
- 收入：余额增加

### 3. 负债/信用类账户计算
- 支出：余额增加（负债增加）✅ 已修复
- 收入：余额减少（负债减少）✅ 已修复

### 4. 转账逻辑
- 现金转信用卡：现金余额减少，信用卡余额减少（负债减少）

### 5. 边界情况
- 零余额账户操作
- 小数精度处理

## 修复逻辑说明

### 负债/信用类账户的特殊处理

对于信用卡、负债账户等，计算逻辑与资产类账户相反：

| 操作类型 | 金额变化 | 余额变化 | 说明 |
|---------|---------|---------|------|
| 支出 | -20 | +20 | 负债增加 |
| 收入/还款 | +100 | -100 | 负债减少 |
| 转账转入 | +50 | -50 | 负债减少 |

### 关键代码
在 `calculateDebtAccountBalance` 函数中：
```javascript
if (isTransferIn) {
  // 转账转入：减少负债
  return (oldBalance - Math.abs(amount)).toFixed(2);
} else {
  // 普通操作需要取反
  return (oldBalance - amount).toFixed(2);
}
```
